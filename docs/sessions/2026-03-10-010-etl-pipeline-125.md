## What was done

- Created `etl/notebooks/pipeline.py` ‚Äî orchestration notebook
  - Uses `dbutils.widgets` for `env` parameter
  - Reads bronze table, calls `clean_orders()` ‚Üí writes silver
  - Reads silver, calls `aggregate_daily_sales()` ‚Üí writes gold
  - Uses `catalog_lookup.get_catalog(env)` for Unity Catalog resolution
  - `saveAsTable(mode="overwrite")` with `overwriteSchema` for idempotency (ADR-003)

- Created `etl/notebooks/e2e_test.py` ‚Äî full E2E test notebook
  - Step 1: Faker generates `num_records` Sales Orders ‚Üí writes to bronze (all STRING columns)
  - Step 2: Runs pipeline inline (clean_orders ‚Üí silver, aggregate_daily_sales ‚Üí gold)
  - Step 3: Schema assertions (Option A) on silver and gold output tables
  - Step 4: Row-count and value assertions (non-empty, gold ‚â§ silver, valid regions, positive revenue)
  - `faker` is a job-level pypi dependency, not in the production wheel

- Created `etl/databricks.yml` ‚Äî Asset Bundles job config
  - `etl-pipeline` job: wheel in libraries, `env` parameter, single-node cluster
  - `etl-e2e-test` job: wheel + `faker` pypi, `env` + `num_records` parameters
  - `dev` (default) and `prod` targets

- Created `data/sample_orders.csv` ‚Äî 10-row static seed for dev bronze seeding

## Design notes

- Bronze table schema is all-STRING (matches raw ingestion pattern); `clean_orders()` handles casting
- `overwriteSchema = true` required because bronze‚Üísilver changes column types
- E2E test uses `Faker.seed(42)` + `random.seed(42)` for reproducible test runs
- `num_records` defaults to 50 in the job; overridable via parameter at run time
